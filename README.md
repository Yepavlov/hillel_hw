Додати можливість видаляти контакт зі списку з обов’язковим підтвердженням через модальне вікно. Реалізацію робити прямо
у файлах уроку.

Функціональні вимоги

1) Кнопка видалення

У кожного контакту в UI має бути кнопка “Delete” / “Remove” (або іконка).
Натискання на кнопку НЕ видаляє контакт одразу.

2) Модальне підтвердження

Після кліку відкривається модальне вікно підтвердження видалення.
Текст модалки: “Ви точно хочете видалити контакт?”
У модалці 2 кнопки:
Confirm (Так, видалити)
Cancel (Скасувати)

3) Поведінка кнопок

Cancel:
закриває модалку
нічого не видаляє
поточний “contact to delete” очищається
Confirm:
видаляє контакт зі state (масив contacts)
видаляє контакт з DOM (зі списку)
закриває модалку
очищає “contact to delete”

4) Заборона “не того видалення”

Поки відкрита модалка, має бути зрозуміло, який саме контакт видаляємо (мінімум: зберігати id/індекс; бажано: показати
ім’я в тексті модалки).
Технічні вимоги

1) Додати у контакт унікальний id

При створенні контакту додайте поле id (наприклад, Date.now() або crypto.randomUUID()).
Цей id має використовуватись для видалення.

2) Доробити contactsManagement

Реалізувати метод removeContact(id):
видаляє контакт з внутрішнього масиву
повертає true/false (успішно/не знайдено) або повертає видалений контакт

3) Прив’язка до DOM

Кожен contact item в DOM повинен мати data-атрибут, наприклад data-id="".
Видалення з DOM робити по цьому id (знайти елемент і прибрати).

4) Події

Використати event delegation (один listener на контейнер списку), або окремий listener на кожну кнопку — на вибір.
Після видалення не повинно залишатись “висячих” обробників (якщо робите без delegation — подумайте про це).
Acceptance Criteria (критерії приймання)

Клік по Delete відкриває модалку підтвердження.
Cancel не змінює список контактів.
Confirm видаляє рівно той контакт, по якому натиснули Delete.
Після видалення контакт зникає і зі state, і з UI.
Повторні операції видалення працюють стабільно (можна видалити кілька контактів по черзі).
Після закриття модалки “contact to delete” очищений.
Додатково (необов’язково, але +)

Показувати в модалці ім’я контакту: “Видалити контакт John Doe?”
Показати toast після успішного видалення: “Контакт видалено”.
Закривати модалку по кліку на backdrop або по Esc.
Підказка по структурі

Вам потрібен тимчасовий стан, наприклад:

let contactIdToDelete = null;
При кліку Delete → contactIdToDelete = id; → openConfirmModal()
Confirm → removeContact(contactIdToDelete) → removeDomNode(contactIdToDelete) → contactIdToDelete = null → closeModal()
Cancel → contactIdToDelete = null → closeModal()
