1. Об’єкт user
   Створіть об’єкт user з такими полями:

name — ім’я
age — вік (number)
email
isSubscribed — boolean
balance — string, наприклад "150.25"
verified — string "1" або "0"
Завдання:
Створіть змінну з числовою версією balance.
Створіть змінну з boolean-версією verified.
На основі даних user визначте, чи має користувач доступ до сервісу. Доступ є, якщо:
вік 18+,
акаунт підтверджений,
є підписка або позитивний баланс.
Порівняйте age і такий самий вік у вигляді рядка через == і через ===. Збережіть результати у дві різні змінні.
Виведіть результат доступу + результати порівнянь.
Якщо користувачу менше 18 — збережіть у змінну текст "Access restricted due to age".

2. Об’єкт order
   Створіть об’єкт order:

total — string, наприклад "950"
currency
isPaid — boolean
delivery — "yes" або "no"
priority — "1" або "0"
Завдання:
Створіть змінну з числовою сумою замовлення.
Створіть змінну з boolean-версією delivery.
Створіть змінну з boolean-версією priority.
Визначте, чи є замовлення великим (від 1000).
Побудуйте текстовий опис замовлення, враховуючи:
не оплачене → "Order is not paid"
оплачене + доставка → "Paid order with delivery"
велике + оплачене → "High-value paid order"
оплачене + без доставки → "Paid order without delivery"
якщо замовлення пріоритетне — додайте " [PRIORITY]"
(Усю логіку будувати лише через if.)
Зробіть два порівняння суми: total == numberTotal та total === numberTotal, запишіть результати у змінні.

3. Об’єкт systemSettings
   Створіть об’єкт systemSettings:

darkMode — boolean
fontSize — string, наприклад "18"
language — "en" або "uk"
betaAccess — string "true" або "false"
Завдання:
Створіть змінну з числовим значенням fontSize.
Створіть змінну з boolean-версією betaAccess.
Створіть змінну isLargeFont (18 і більше).
Побудуйте текст налаштувань (через if):
темна тема + великий шрифт → "Dark mode + large font"
темна тема → "Dark mode"
великий шрифт → "Large font"
інакше → "Default settings"
Якщо користувач має бета-доступ — додайте " (Beta tester)".

4. Інтеграція: фінальний доступ до системи
   На основі ВСІХ трьох об’єктів створіть змінну finalAccess, яка визначає, чи користувач має повний доступ до сервісу.

Повний доступ дозволено тільки якщо:

доступ з блоку user дозволений
замовлення оплачене або у користувача достатньо грошей на балансі, щоб покрити суму замовлення
валідні налаштування системи:
шрифт > 12
мова "en" або "uk"
Усі перевірки виконуються лише через if + логічні оператори.

У консоль потрібно:

"Full access granted" або "Access denied"
окремо текст, яка умова заблокувала доступ (user/order/system).
(Використовуйте окремі змінні для проміжних результатів перевірок.)
